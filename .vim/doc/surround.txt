              <div class="highlight"><pre><div class='line' id='LC1'>*surround.txt*  Plugin for deleting, changing, and adding "surroundings"</div><div class='line' id='LC2'><br/></div><div class='line' id='LC3'>Author:  Tim Pope &lt;vimNOSPAM@tpope.info&gt;        *surround-author*</div><div class='line' id='LC4'>License: Same terms as Vim itself (see |license|)</div><div class='line' id='LC5'><br/></div><div class='line' id='LC6'>This plugin is only available if &#39;compatible&#39; is not set.</div><div class='line' id='LC7'><br/></div><div class='line' id='LC8'>INTRODUCTION                                    *surround*</div><div class='line' id='LC9'><br/></div><div class='line' id='LC10'>This plugin is a tool for dealing with pairs of "surroundings."  Examples</div><div class='line' id='LC11'>of surroundings include parentheses, quotes, and HTML tags.  They are</div><div class='line' id='LC12'>closely related to what Vim refers to as |text-objects|.  Provided</div><div class='line' id='LC13'>are mappings to allow for removing, changing, and adding surroundings.</div><div class='line' id='LC14'><br/></div><div class='line' id='LC15'>Details follow on the exact semantics, but first, consider the following</div><div class='line' id='LC16'>examples.  An asterisk (*) is used to denote the cursor position.</div><div class='line' id='LC17'><br/></div><div class='line' id='LC18'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC19'>&nbsp;&nbsp;"Hello *world!"           ds"         Hello world!</div><div class='line' id='LC20'>&nbsp;&nbsp;[123+4*56]/2              cs])        (123+456)/2</div><div class='line' id='LC21'>&nbsp;&nbsp;"Look ma, I&#39;m *HTML!"     cs"&lt;q&gt;      &lt;q&gt;Look ma, I&#39;m HTML!&lt;/q&gt;</div><div class='line' id='LC22'>&nbsp;&nbsp;if *x&gt;3 {                 ysW(        if ( x&gt;3 ) {</div><div class='line' id='LC23'>&nbsp;&nbsp;my $str = *whee!;         vlllls&#39;     my $str = &#39;whee!&#39;;</div><div class='line' id='LC24'><br/></div><div class='line' id='LC25'>While a few features of this plugin will work in older versions of Vim,</div><div class='line' id='LC26'>Vim 7 is recommended for full functionality.</div><div class='line' id='LC27'><br/></div><div class='line' id='LC28'>MAPPINGS                                        *surround-mappings*</div><div class='line' id='LC29'><br/></div><div class='line' id='LC30'>Delete surroundings is *ds* .  The next character given determines the target</div><div class='line' id='LC31'>to delete.  The exact nature of the target is explained in |surround-targets|</div><div class='line' id='LC32'>but essentially it is the last character of a |text-object|.  This mapping</div><div class='line' id='LC33'>deletes the difference between the "i"nner object and "a"n object.  This is</div><div class='line' id='LC34'>easiest to understand with some examples:</div><div class='line' id='LC35'><br/></div><div class='line' id='LC36'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC37'>&nbsp;&nbsp;"Hello *world!"           ds"         Hello world!</div><div class='line' id='LC38'>&nbsp;&nbsp;(123+4*56)/2              ds)         123+456/2</div><div class='line' id='LC39'>&nbsp;&nbsp;&lt;div&gt;Yo!*&lt;/div&gt;           dst         Yo!</div><div class='line' id='LC40'><br/></div><div class='line' id='LC41'>Change surroundings is *cs* .  It takes two arguments, a target like with</div><div class='line' id='LC42'>|ds|, and a replacement.  Details about the second argument can be found</div><div class='line' id='LC43'>below in |surround-replacements|.  Once again, examples are in order.</div><div class='line' id='LC44'><br/></div><div class='line' id='LC45'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC46'>&nbsp;&nbsp;"Hello *world!"           cs"&#39;        &#39;Hello world!&#39;</div><div class='line' id='LC47'>&nbsp;&nbsp;"Hello *world!"           cs"&lt;q&gt;      &lt;q&gt;Hello world!&lt;/q&gt;</div><div class='line' id='LC48'>&nbsp;&nbsp;(123+4*56)/2              cs)]        [123+456]/2</div><div class='line' id='LC49'>&nbsp;&nbsp;(123+4*56)/2              cs)[        [ 123+456 ]/2</div><div class='line' id='LC50'>&nbsp;&nbsp;&lt;div&gt;Yo!*&lt;/div&gt;           cst&lt;p&gt;      &lt;p&gt;Yo!&lt;/p&gt;</div><div class='line' id='LC51'><br/></div><div class='line' id='LC52'>*ys* takes a valid Vim motion or text object as the first object, and wraps</div><div class='line' id='LC53'>it using the second argument as with |cs|.  (It&#39;s a stretch, but a good</div><div class='line' id='LC54'>mnemonic for "ys" is "you surround".)</div><div class='line' id='LC55'><br/></div><div class='line' id='LC56'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC57'>&nbsp;&nbsp;Hello w*orld!             ysiw)       Hello (world)!</div><div class='line' id='LC58'><br/></div><div class='line' id='LC59'>As a special case, *yss* operates on the current line, ignoring leading</div><div class='line' id='LC60'>whitespace.</div><div class='line' id='LC61'><br/></div><div class='line' id='LC62'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC63'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;Hello w*orld!         yssB            {Hello world!}</div><div class='line' id='LC64'><br/></div><div class='line' id='LC65'>There is also *yS* and *ySS* which indent the surrounded text and place it</div><div class='line' id='LC66'>on a line of its own.</div><div class='line' id='LC67'><br/></div><div class='line' id='LC68'>In visual mode, a simple "s" with an argument wraps the selection.  This is</div><div class='line' id='LC69'>referred to as the *vS* mapping, although ordinarily there will be</div><div class='line' id='LC70'>additional keystrokes between the v and s.  In linewise visual mode, the</div><div class='line' id='LC71'>surroundings are placed on separate lines and indented.  In blockwise visual</div><div class='line' id='LC72'>mode, each line is surrounded.</div><div class='line' id='LC73'><br/></div><div class='line' id='LC74'>A "gS" in visual mode, known as *vgS* , behaves similarly.  In linewise visual</div><div class='line' id='LC75'>mode, the automatic indenting is surpressed.  In blockwise visual mode, this</div><div class='line' id='LC76'>enables surrounding past the end of the like with &#39;virtualedit&#39; set (there</div><div class='line' id='LC77'>seems to be no way in Vim Script to differentiate between a jagged end of line</div><div class='line' id='LC78'>selection and a virtual block selected past the end of the line, so two maps</div><div class='line' id='LC79'>were needed).</div><div class='line' id='LC80'><br/></div><div class='line' id='LC81'>Additionally, there is a legacy "s" or *vs* mapping which is basically the</div><div class='line' id='LC82'>same as |vS|.  Due to popular demand of wanting to use "s" as Vim does to mean</div><div class='line' id='LC83'>replacing the selection (also available as "c"), this mapping is going away.</div><div class='line' id='LC84'>If you were one of these people and would like to disable "s" with the current</div><div class='line' id='LC85'>release, indicate this to surround.vim by assigning the "s" mapping to</div><div class='line' id='LC86'>something else.</div><div class='line' id='LC87'>&gt;</div><div class='line' id='LC88'>&nbsp;&nbsp;xmap &lt;Leader&gt;s &lt;Plug&gt;Vsurround</div><div class='line' id='LC89'>&lt;</div><div class='line' id='LC90'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*i_CTRL-G_s* *i_CTRL-G_S*</div><div class='line' id='LC91'>Finally, there is an experimental insert mode mapping on &lt;C-G&gt;s and &lt;C-S&gt;.</div><div class='line' id='LC92'>Beware that the latter won&#39;t work on terminals with flow control (if you</div><div class='line' id='LC93'>accidentally freeze your terminal, use &lt;C-Q&gt; to unfreeze it).  The mapping</div><div class='line' id='LC94'>inserts the specified surroundings and puts the cursor between them.  If,</div><div class='line' id='LC95'>immediately after the mapping and before the replacement, a second &lt;C-S&gt; or</div><div class='line' id='LC96'>carriage return is pressed, the prefix, cursor, and suffix will be placed on</div><div class='line' id='LC97'>three separate lines.  &lt;C-G&gt;S (not &lt;C-G&gt;s) also exhibits this behavior.</div><div class='line' id='LC98'><br/></div><div class='line' id='LC99'>TARGETS                                         *surround-targets*</div><div class='line' id='LC100'><br/></div><div class='line' id='LC101'>The |ds| and |cs| commands both take a target as their first argument.  The</div><div class='line' id='LC102'>possible targets are based closely on the |text-objects| provided by Vim.</div><div class='line' id='LC103'>In order for a target to work, the corresponding text object must be</div><div class='line' id='LC104'>supported in the version of Vim used (Vim 7 adds several text objects, and</div><div class='line' id='LC105'>thus is highly recommended).  All targets are currently just one character.</div><div class='line' id='LC106'><br/></div><div class='line' id='LC107'>Eight punctuation marks, (, ), {, }, [, ], &lt;, and &gt;, represent themselves</div><div class='line' id='LC108'>and their counterparts.  If the opening mark is used, contained whitespace is</div><div class='line' id='LC109'>also trimmed.  The targets b, B, r, and a are aliases for ), }, ], and &gt;</div><div class='line' id='LC110'>(the first two mirror Vim; the second two are completely arbitrary and</div><div class='line' id='LC111'>subject to change).</div><div class='line' id='LC112'><br/></div><div class='line' id='LC113'>Three quote marks, &#39;, ", `, represent themselves, in pairs.  They are only</div><div class='line' id='LC114'>searched for on the current line.</div><div class='line' id='LC115'><br/></div><div class='line' id='LC116'>A t is a pair of HTML or XML tags.  See |tag-blocks| for details.  Remember</div><div class='line' id='LC117'>that you can specify a numerical argument if you want to get to a tag other</div><div class='line' id='LC118'>than the innermost one.</div><div class='line' id='LC119'><br/></div><div class='line' id='LC120'>The letters w, W, and s correspond to a |word|, a |WORD|, and a |sentence|,</div><div class='line' id='LC121'>respectively.  These are special in that they have nothing to delete, and</div><div class='line' id='LC122'>used with |ds| they are a no-op.  With |cs|, one could consider them a</div><div class='line' id='LC123'>slight shortcut for ysi (cswb == ysiwb, more or less).</div><div class='line' id='LC124'><br/></div><div class='line' id='LC125'>A p represents a |paragraph|.  This behaves similarly to w, W, and s above;</div><div class='line' id='LC126'>however, newlines are sometimes added and/or removed.</div><div class='line' id='LC127'><br/></div><div class='line' id='LC128'>REPLACEMENTS                                    *surround-replacements*</div><div class='line' id='LC129'><br/></div><div class='line' id='LC130'>A replacement argument is a single character, and is required by |cs|, |ys|,</div><div class='line' id='LC131'>and |vs|.  Undefined replacement characters (with the exception of alphabetic</div><div class='line' id='LC132'>characters) default to placing themselves at the beginning and end of the</div><div class='line' id='LC133'>destination, which can be useful for characters like / and |.</div><div class='line' id='LC134'><br/></div><div class='line' id='LC135'>If either ), }, ], or &gt; is used, the text is wrapped in the appropriate pair</div><div class='line' id='LC136'>of characters.  Similar behavior can be found with (, {, and [ (but not &lt;),</div><div class='line' id='LC137'>which append an additional space to the inside.  Like with the targets above,</div><div class='line' id='LC138'>b, B, r, and a are aliases for ), }, ], and &gt;.  To fulfill the common need for</div><div class='line' id='LC139'>code blocks in C-style languages, &lt;C-}&gt; (which is really &lt;C-]&gt;) adds braces on</div><div class='line' id='LC140'>lines separate from the content.</div><div class='line' id='LC141'><br/></div><div class='line' id='LC142'>If t or &lt; is used, Vim prompts for an HTML/XML tag to insert.  You may specify</div><div class='line' id='LC143'>attributes here and they will be stripped from the closing tag.  End your</div><div class='line' id='LC144'>input by pressing &lt;CR&gt; or &gt;.  If &lt;C-T&gt; is used, the tags will appear on lines</div><div class='line' id='LC145'>by themselves.</div><div class='line' id='LC146'><br/></div><div class='line' id='LC147'>A deprecated replacement of a LaTeX environment is provided on \ and l.  The</div><div class='line' id='LC148'>name of the environment and any arguments will be input from a prompt.  This</div><div class='line' id='LC149'>will be removed once a more fully functional customization system is</div><div class='line' id='LC150'>implemented.  The following shows the resulting environment from</div><div class='line' id='LC151'>csp\tabular}{lc&lt;CR&gt;</div><div class='line' id='LC152'>&gt;</div><div class='line' id='LC153'>&nbsp;&nbsp;\begin{tabular}{lc}</div><div class='line' id='LC154'>&nbsp;&nbsp;\end{tabular}</div><div class='line' id='LC155'>&lt;</div><div class='line' id='LC156'>CUSTOMIZING                                     *surround-customizing*</div><div class='line' id='LC157'><br/></div><div class='line' id='LC158'>The following adds a potential replacement on "-" (ASCII 45) in PHP files.</div><div class='line' id='LC159'>(To determine the ASCII code to use, :echo char2nr("-")).  The carriage</div><div class='line' id='LC160'>return will be replaced by the original text.</div><div class='line' id='LC161'>&gt;</div><div class='line' id='LC162'>&nbsp;&nbsp;autocmd FileType php let b:surround_45 = "&lt;?php \r ?&gt;"</div><div class='line' id='LC163'>&lt;</div><div class='line' id='LC164'>This can be used in a PHP file as in the following example.</div><div class='line' id='LC165'><br/></div><div class='line' id='LC166'>&nbsp;&nbsp;Old text                  Command     New text ~</div><div class='line' id='LC167'>&nbsp;&nbsp;print "Hello *world!"     yss-        &lt;?php print "Hello world!" ?&gt;</div><div class='line' id='LC168'><br/></div><div class='line' id='LC169'>Additionally, one can use a global variable for globally available</div><div class='line' id='LC170'>replacements.</div><div class='line' id='LC171'>&gt;</div><div class='line' id='LC172'>&nbsp;&nbsp;let g:surround_45 = "&lt;% \r %&gt;"</div><div class='line' id='LC173'>&nbsp;&nbsp;let g:surround_61 = "&lt;%= \r %&gt;"</div><div class='line' id='LC174'>&lt;</div><div class='line' id='LC175'>Advanced, experimental, and subject to change:  One can also prompt for</div><div class='line' id='LC176'>replacement text.  The syntax for this is to surround the replacement in pairs</div><div class='line' id='LC177'>of low numbered control characters.  If this sounds confusing, that&#39;s because</div><div class='line' id='LC178'>it is (but it makes the parsing easy).  Consider the following example for a</div><div class='line' id='LC179'>LaTeX environment on the "l" replacement.</div><div class='line' id='LC180'>&gt;</div><div class='line' id='LC181'>&nbsp;&nbsp;let g:surround_108 = "\\begin{\1environment: \1}\r\\end{\1\1}"</div><div class='line' id='LC182'>&lt;</div><div class='line' id='LC183'>When this replacement is used,  the user is prompted with an "environment: "</div><div class='line' id='LC184'>prompt for input.  This input is inserted between each set of \1&#39;s.</div><div class='line' id='LC185'>Additional inputs up to \7 can be used.</div><div class='line' id='LC186'><br/></div><div class='line' id='LC187'>Furthermore, one can specify a regular expression substitution to apply.</div><div class='line' id='LC188'>&gt;</div><div class='line' id='LC189'>&nbsp;&nbsp;let g:surround_108 = "\\begin{\1environment: \1}\r\\end{\1\r}.*\r\1}"</div><div class='line' id='LC190'>&lt;</div><div class='line' id='LC191'>This will remove anything after the first } in the input when the text is</div><div class='line' id='LC192'>placed within the \end{} slot.  The first \r marks where the pattern begins,</div><div class='line' id='LC193'>and the second where the replacement text begins.</div><div class='line' id='LC194'><br/></div><div class='line' id='LC195'>Here&#39;s a second example for creating an HTML &lt;div&gt;.  The substitution cleverly</div><div class='line' id='LC196'>prompts for an id, but only adds id="" if it is non-blank.  You may have to</div><div class='line' id='LC197'>read this one a few times slowly before you understand it.</div><div class='line' id='LC198'>&gt;</div><div class='line' id='LC199'>&nbsp;&nbsp;let g:surround_{char2nr("d")} = "&lt;div\1id: \r..*\r id=\"&amp;\"\1&gt;\r&lt;/div&gt;"</div><div class='line' id='LC200'>&lt;</div><div class='line' id='LC201'>Inputting text replacements is a proof of concept at this point.
